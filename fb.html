别把AI用成付费上班

够用原则、分层路由、精准交付——把AI账单用在刀刃上，省钱只是结果，清醒才是内核

嵌入式AI工程化系列 · 工程心法

核心结论

旗舰模型在系统性软件工程任务上，确实更强——这是事实，不用嘴硬。但你日常百分之八十的活，根本用不到旗舰。同样的活，国产模型一样能干完，价格只要十分之一甚至更低。前提是你要会用。不会用，给你最贵的模型也是浪费；会用，DeepSeek、Kimi、GLM、Qwen一样能把你从加班里捞出来。

一、两千美元的账单，买的是什么

我看到一个数据：有工程师月花近两千美元调用AI的API。不是搞AI创业，就是日常项目开发、改Bug、整理文档。网友调侃说，这不叫用AI，这叫付费上班。两千美元是什么概念？按现在的汇率，差不多一万四千块人民币。一个月的AI开销，顶得上很多人的月薪。

但如果你拉出他的对话记录看，会发现百分之八十以上的请求——改CRUD、写单测、改Lint、整理文档、写周报——中端模型甚至轻量模型就能完美处理。他不是被AI替代了，他是被自己的"旗舰焦虑"替代了。

月花两千美元的人，大多数没多写一行真正复杂的业务代码，他们只是把"用最强的模型"当成心理安慰。这种安慰，你的公司不会买单，你的钱包也不会买单。

二、够用原则：不是最强的，是最合适的

我在技术文章里反复写过一个词：够用原则。问题解决了，就不要再继续改。版本能跑了，就不要再追求完美。AI的输出已经能满足需求了，就不要再反复调参数。这条原则放在选模型上，同样适用。

很多人心里有个偏见：国产模型不行。这个偏见在二零二四年可能还成立，到了二零二六年，在大多数日常任务上已经不成立了。我不替国产护短——复杂架构设计、跨文件重构，头部旗舰确实还领先。但你想想，你工作里这类任务占多少比例？我自己复盘过，大概只占百分之十五到百分之二十。剩下百分之八十是改Bug、写单测、写文档、改Lint、简单接口对接。这些任务，旗舰和中端模型你盲测大概率分不出来。

够用原则回顾

此前在《君子不器与灵活调度》和《批量修改，先问该不该用脚本》中都探讨过核心思路：优先做判断，再选择工具，不为追逐新潮而滥用工具。选模型和选工具是同一个逻辑——不是选最强的，是选最对的。

三、分层路由：不是选最强的，是选最对的

我在工程里一直用一套逻辑：分层。不是所有任务都值得调用最强的模型，就像不是所有代码都值得用汇编去写。把任务按复杂度分档，然后匹配对应档位的模型。

日常任务占八成，用国产中端，响应往往更快，成本只有海外的十分之一。中等复杂度任务占一成半，用海外中端或国产长上下文。真正啃硬骨头的时候，只占半成，才请出海外旗舰。

如果你把这个逻辑写成一段自动化路由的伪代码，它大概长这样：

def route_task(task):
    if task.complexity == "simple":
        return "国产中端"
    elif task.complexity == "medium":
        return "海外中端 or 国产长上下文"
    elif task.complexity == "hard":
        return "海外旗舰"

按这个分层走，普通工程师的API账单能从月花上千美元压到几十块到几百块人民币。输出量不掉，响应更快，有时候还省时间。省下来的不是抠门，是判断力。是你知道什么时候该用重武器，什么时候一把螺丝刀就够了。

🟢 日常任务 · 80%

改Bug、写单测、改Lint、整理文档、写周报——国产中端模型即可完美处理

🟡 中等任务 · 15%

跨模块调试、长上下文理解、SQL优化——海外中端或国产长上下文

🔴 硬核任务 · 5%

复杂架构设计、跨文件重构、系统性重构——海外旗舰模型

💡 核心逻辑

不是选最强的，是选最对的。判断力决定性价比，而非模型参数

四、真正的护城河不是模型，是你自己

我见过太多人，给最强的模型也写不好代码。他们的问题不是模型不够强，是自己不会用。一句话指令——"帮我写个登录功能"，再贵的模型也救不了你。不喂上下文——不贴现有代码、不说技术栈，模型只能瞎猜。一锅炖几十个需求，超长提示词丢进去，模型理解错一半。错了不教，只会重新生成，而不是告诉它错在哪、应该怎么改。

这四个毛病不改，旗舰和中端在你手里输出的差距，可能不到一成。反过来，提示词写清楚、上下文喂够、反馈做精准，国产中端一样能给你九十分的结果。旗舰的优势是容错率高——它能在你提示词写得很烂的情况下也猜对意图。但这不是你应该花十几倍价钱去买的东西。

护城河三件套

把那十几倍预算，用来训练自己写好提示词，用来搭建自己的上下文管理流程，用来学会怎么给AI精准的反馈。这才是你真正的护城河。模型会迭代，插件会过时，但判断力永远不会贬值。

旗舰的优势是容错率高——它能在你提示词写得很烂的情况下也猜对意图。但你应该做的不是花十几倍价钱买容错，而是把自己的提示词写到不需要容错。

五、四步找到你自己的性价比模型

不要盲信任何博主推荐，包括我。你的工作场景只有你自己最清楚。

  1. 列出过去一周的AI任务清单。
  2. 按复杂度分三档——简单任务如改Bug、写注释、翻译；中等任务如写单测、写文档、SQL优化；复杂任务如跨文件重构、架构设计。
  3. 每档挑几个任务，用不同档位的模型盲测，遮掉模型名自己打分。你会发现简单档和中等档的差距小到你不愿意相信。
  4. 按你的真实任务分布算成本，你就知道钱该花在哪。这是唯一不会骗你的数据，因为是你自己跑出来的。

行动指南

不要看别人用什么你就用什么。花一个小时盲测，跑出来的数据会比任何博主的推荐更诚实。你的工作场景只有你自己最清楚，你的性价比模型也只有你自己能定义。

六、结语：清醒才是内核

你最该追的不是全球最强那块奖牌，而是对你来说最划算那张账单。旗舰不是不能用，是不能无脑只用旗舰。国产不是替代品，是你百分之八十工作场景里的主力。那些月花上千美元的人，大多数没多写一行真正复杂的业务代码，他们只是把"用最强的模型"当成心理安慰。

真正能让你不被AI取代的，从来不是"我用了最贵的模型"，而是我知道每个任务该用什么模型，以及怎么把它用到极致。

够用原则，分层路由，精准交付。这和"君子不器"讲的是同一件事——对内，不把自己活成只会调旗舰的工具；对外，不沉迷于堆砌最贵的模型。守住人的判断力，把工具用在刀刃上。省钱只是结果，清醒才是内核。

本文定位

本文是嵌入式AI工程化系列的工程心法篇。它从"付费上班"这一网络调侃切入，探讨AI模型选择中的够用原则与分层路由策略。核心观点包括：旗舰焦虑（被心理安慰驱动的无效支出）、够用原则（选最对的而非最强的）、分层路由（按任务复杂度匹配模型档位）、护城河在自己（提示词能力 > 模型参数）。与《君子不器与灵活调度》互为补充，前者讲工具调度法则，本文讲模型选择经济学。建议在阅读 lj.html 后阅读，但可独立使用。
