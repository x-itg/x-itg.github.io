## 🧭 站点坐标

这个站点不是纯技术手册，也不是生活随笔合集，而是把一套工程法则从物理世界一路推演到生活、小说与创作方法的四层结构。

- **没有“全面碾压”的同类型站点**：但在纯技术深度、行业影响力、长期沉淀与跨域融合成熟度上，确实有一批值得长期对标的优秀独立博客。
- **技术深度的对标方向**：痞子衡、Embedded Artistry、Interrupt Blog 这类站点，代表了底层工具链、固件架构、调试与可观测性的高标准。
- **技术与人文的对标方向**：小土刀 2.0 代表的是更成熟的长文思辨密度与克制文风。
- **元创作的对标方向**：陈素封提醒我，创作过程本身也能被提炼成长期方法论。
- **这座站点真正独特的部分**：硬件淤青 → 工程心法 → 生活关系 → 小说实验 → 元创作公开，这条完整链路目前仍然很少见。

## 🧠 核心哲学

- **意图定义者 > 代码执行者**：人的核心价值从"编写指令"跃迁为"定义约束和边界"。
- **可观测性即权利**：结构化日志是 AI 在物理世界中的"眼睛"；正日志记录"发生了什么"，负日志捕获"应该发生却未发生"。
- **闭环是思维，契约是边界**：每一次修改必须有新日志驱动；用提示词模板和规约文件限制 AI 的输出空间。
- **共生而非替代**：人保有最终裁决权，AI 负责追踪分歧、展示后果、请求决策。
- **法则从实践中来**：真正的规律不是从书本到书本，而是从实践到理论——从淤青中长出铠甲。

## ⚡ 大侠，这套心法怎么练？

1. **初窥门径**：先读 ⑬ 利刃，明天就试试让 AI 自己写测试、自己改。
2. **小有所成**：搭起 ② 模板，建起你的法则文件夹，用 ③ 闭环收敛打磨每个模块。
3. **游历江湖**：碰到老项目用 ⑧，想把 AI 接进真实硬件调试先读 ⑨，多设备管理用 ⑩，合规审计用 ⑪。
4. **破境冲关**：先读 ⑭，看批量改造时如何选对执行模式；再用 ⑮ 拆解整合驯服复杂系统，用 ④ 决策切换避免死胡同。
5. **专业守关**：读 ⑯ AI 的谄媚与专利，把“顺着写”和“审查怼”做成可切换流程。
6. **回望内核**：读 ⑰《始于工程，服务生活》，看这套工程法则为什么最终会穿过技术、落回人生。
7. **一代宗师**：言出法随，即刻收敛，踏入五层进化之巅。

## 🧩 分层点评模板

为了让 L1-L4 页面后续追加点评时不再重复拷贝局部样式，站点现在统一提供了共享样式文件 `assets/article-comments.css`。

- `layer-comments layer-comments--review`：L1 用，少量、理性、像评注。
- `layer-comments layer-comments--excerpt`：L2 用，少量对话摘录，保留活人感但不做热闹群聊。
- `layer-comments layer-comments--chat`：L3/L4 用，完整群聊/评论窗口，更适合实验页和方法页。

新增页面时只需要在页头额外引入一次：`<link rel="stylesheet" href="../assets/article-comments.css">`，再按层级选择对应的 HTML 结构即可。页面自身继续保留内容语气差异，但点评区的视觉骨架不再分散维护。
